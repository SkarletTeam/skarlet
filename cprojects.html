Automated Risk Reporting Scripts

During my previous role, I developed a suite of reporting scripts that enabled the efficient extraction of data from various sources, including vulnerability management software, Active Directory (AD), and other relevant systems. These scripts were designed to calculate risk based on the current environment, providing actionable insights for informed decision-making.

The scripts successfully aggregated data from disparate sources, analyzed the information, and generated comprehensive risk reports. This automation not only streamlined the reporting process but also ensured accuracy and consistency in risk assessment.

Key Benefits:

  • Automated data extraction from multiple sources
  • Calculated risk based on current environment
  • Generated comprehensive risk reports
  • Improved accuracy and consistency in risk assessment
  • Enhanced decision-making capabilities


Python Script for Automated Documentation

I developed a Python script that processes data from a spreadsheet and populates it into a Word document, similar to the functionality of mail merge. The script is designed to enhance the efficiency of the documentation process for security reporting.

Key Features:

  • Data Parsing: The script reads the spreadsheet and extracts relevant data.
  • Table Population: Certain fields can populate tables if the entries are separated by commas.
  • Efficiency: This automation significantly speeds up the documentation process.

Please note that the dummy data used in the script is not sensitive, so it can be handled as needed.

You can find the GitHub repository for this project MailMerge.
